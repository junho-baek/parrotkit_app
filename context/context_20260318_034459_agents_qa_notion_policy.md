# Context - AGENTS QA / Notion 기본 정책 변경

## 작업 배경
- 사용자 요청: 앞으로는 기본 검증을 `build`가 아니라 `dev` 기준 QA로 진행하고 싶다는 요청.
- 추가 요청: Notion 업로드는 자동 기본값이 아니라 사용자가 명시적으로 요청할 때만 수행하도록 바꾸고 싶다는 요청.

## 변경 목표
- `AGENTS.md`에 `dev` 기반 QA를 기본 검증 흐름으로 명시.
- `npm run build`는 사용자 요청 시에만 실행하도록 규칙 변경.
- Notion 업로드는 사용자 요청 시에만 수행하도록 규칙 변경.

## 변경 내용
- 파일: `AGENTS.md`
  - Orchestrator 규칙의 3번 항목을 `dev` 기준 QA 기본값으로 수정
  - `build`는 사용자 명시 요청 시에만 실행하도록 명시
  - GitHub push 전 Notion 업로드 의무 문구를 제거하고 사용자 요청 기반으로 수정
  - 작업 절차 5번 항목을 `dev` 기준 검증 문구로 수정
  - Notion 리포트 자동화 규칙을 “사용자 요청 시에만 업로드” 흐름으로 수정
  - 배포 QA 규칙의 Notion 업로드 의무 문구를 사용자 요청 기반으로 수정
- 파일: `plans/20260318_agents_qa_notion_policy.md`
  - 작업 계획 문서 작성

## 검증
- 문서 정책 변경 작업으로 별도 `build` / runtime QA / Notion 업로드는 수행하지 않음
- 이유: 이번 변경 자체가 “build를 기본값으로 두지 않는다”와 “Notion 업로드는 사용자 요청 시에만 수행한다”는 정책 업데이트이기 때문

## 기대 동작
- 앞으로 기본 로컬 검증은 `dev` 기준 QA로 진행한다.
- `npm run build`는 사용자가 명시적으로 요청한 경우에만 실행한다.
- Notion 업로드는 사용자가 명시적으로 요청한 경우에만 수행한다.
