# Parrotkit GA4 UTM 확인 가이드

## 문서 목적
- 외부 링크에 붙인 UTM 파라미터가 GA4에서 정상적으로 잡히는지 확인하는 방법을 한국어로 정리한다.
- 실제로 확인된 예시를 바탕으로, `세션 소스/매체`, `세션 캠페인`, 핵심 이벤트 전환을 어디서 읽어야 하는지 설명한다.
- 마케팅 실험에 바로 사용할 수 있는 링크 운영 원칙과 기대효과를 정리한다.

## 한 줄 결론
- Parrotkit 배포환경에서 UTM은 정상적으로 작동한다.
- 실제 확인 예시:
  - `세션 소스/매체 = notion / docs`
  - `세션 캠페인 = pricing_test`
  - GA4 실시간에서 `purchase_success`까지 확인됨

## 왜 이 문서를 봐야 하나
GA4는 기본적으로 UTM 파라미터를 읽어 캠페인 유입을 분류한다. 하지만 화면이 여러 개로 나뉘어 있어서 처음 보면 어디서 무엇을 확인해야 할지 헷갈리기 쉽다. Parrotkit에서는 아래 순서로 보면 가장 빠르다.

1. `트래픽 획득`에서 UTM 유입이 들어왔는지 본다.
2. `실시간 개요` 또는 `DebugView`에서 이벤트가 실제로 찍혔는지 본다.
3. `탐색`에서 유입과 이벤트 전환을 한 표로 본다.

## 실제 확인 예시
이번 검증에서 실제로 확인된 값은 아래와 같다.

### 예시 1. UTM 유입 확인
- 확인 위치: `보고서 -> 획득 -> 트래픽 획득`
- 차원 설정:
  - 기본 차원: `세션 소스/매체`
  - 보조 차원: `세션 캠페인`
- 실제 확인 값:
  - `notion / docs`
  - `pricing_test`

이 값이 보인다는 뜻은 아래 링크가 GA4에 의도한 형태로 들어왔다는 의미다.

```plain text
https://parrotkit-deploy.vercel.app/pricing?utm_source=notion&utm_medium=docs&utm_campaign=pricing_test
```

### 예시 2. 이벤트 확인
- 확인 위치: `실시간 개요`
- 실제 확인 이벤트:
  - `view_pricing`
  - `begin_checkout`
  - `purchase_success`

즉, UTM으로 들어온 사용자가 Pricing에 도달했고, 결제를 시작했고, 최종적으로 성공 이벤트까지 기록된 것을 확인한 상태다.

## GA4에서 UTM 확인하는 가장 쉬운 방법

### 1. 트래픽 획득에서 유입 확인
경로:
- `보고서`
- `획득`
- `트래픽 획득`

설정:
1. 표 첫 번째 열의 차원을 `세션 소스/매체`로 바꾼다.
2. 옆에 보조 차원으로 `세션 캠페인`을 추가한다.

정상 예시:
- `notion / docs`
- `pricing_test`

이 값이 보이면 UTM이 GA4에 정상 반영된 것이다.

### 2. 실시간에서 이벤트 확인
경로:
- `보고서`
- `실시간 개요`

여기서 확인할 것:
- `view_pricing`
- `begin_checkout`
- `purchase_success`
- 필요 시
  - `login`
  - `signup_success`
  - `onboarding_complete`
  - `recipe_generated`
  - `recipe_saved`

실시간 화면은 “방금 들어온 유입이 실제 행동까지 이어졌는가”를 보는 데 가장 빠르다.

### 3. 탐색에서 유입 -> 이벤트 전환 보기
경로:
- `탐색`
- `자유 형식`

권장 리포트 이름:
- `UTM -> Event Conversion`

추가할 차원:
- `세션 소스/매체`
- `세션 캠페인`
- `이벤트 이름`

추가할 측정항목:
- `이벤트 수`
- `총 사용자`

탭 설정:
- 행:
  - `세션 소스/매체`
  - `세션 캠페인`
- 열:
  - `이벤트 이름`
- 값:
  - `이벤트 수`

추천 필터:
- `세션 캠페인 = pricing_test`
- `이벤트 이름` 포함:
  - `login`
  - `view_pricing`
  - `begin_checkout`
  - `purchase_success`

이렇게 보면 “어떤 UTM으로 들어온 사용자가 어떤 이벤트까지 갔는지”를 한 표에서 바로 읽을 수 있다.

## UTM 링크는 어디에 붙이나
UTM은 광고에만 붙이는 것이 아니다. Parrotkit에서는 외부에서 앱으로 유입시키는 거의 모든 링크에 붙일 수 있다.

붙일 수 있는 곳:
- Notion 공개 문서 CTA
- 이메일 뉴스레터
- 커뮤니티 글
- 인스타 프로필 링크
- 파트너 소개 페이지
- Meta 광고
- Google 광고
- QR 코드 링크

붙이면 안 되는 곳:
- 앱 내부 네비게이션
- `/pricing`에서 `/my`로 가는 내부 링크
- 이미 앱에 들어온 뒤의 탭 이동

원칙:
- **외부 -> 앱 진입 링크에만 UTM을 붙인다.**

## Parrotkit에서 바로 할 수 있는 마케팅

### 1. Notion 문서 기반 유입 테스트
예시:
- `utm_source=notion`
- `utm_medium=docs`
- `utm_campaign=pricing_test`

활용:
- 팀 소개 문서, 제품 소개 문서, 제안서에 CTA를 넣고 실제 유입을 분리한다.

기대효과:
- “노션 문서에서 실제 사용자가 들어오는가”를 감으로 보지 않고 숫자로 본다.
- 문서 버전별 CTA 성과를 비교할 수 있다.

### 2. 커뮤니티/콘텐츠 유입 비교
예시:
- `utm_source=community`
- `utm_medium=organic_post`
- `utm_campaign=beta_launch`

활용:
- 커뮤니티 글 제목, 썸네일, 카피를 바꿔 유입과 전환을 비교한다.

기대효과:
- 어느 채널이 단순 방문만 만드는지, 어느 채널이 결제까지 이어지는지 구분할 수 있다.

### 3. Pricing 직행 실험
예시:
- `utm_source=meta`
- `utm_medium=paid_social`
- `utm_campaign=pricing_test`

활용:
- 랜딩 홈이 아니라 Pricing으로 바로 보내는 실험을 할 수 있다.

기대효과:
- 구매 의도가 높은 유입에서 checkout 전환율을 빠르게 볼 수 있다.
- 광고 메시지와 Pricing 카피의 정합성을 검증할 수 있다.

### 4. 퍼널 병목 확인
UTM 유입 후 아래 퍼널을 본다.
- `view_pricing`
- `begin_checkout`
- `purchase_success`

기대효과:
- Pricing 페이지 문제인지
- 결제 진입 버튼 문제인지
- 체크아웃 이탈 문제인지
구간별로 나눠서 볼 수 있다.

## 기대효과 요약
- 어떤 외부 채널이 실제 유입을 만드는지 확인 가능
- 어떤 캠페인이 결제까지 이어지는지 추적 가능
- 노션/커뮤니티/뉴스레터처럼 광고비가 들지 않는 채널의 기여도 측정 가능
- Meta/Google 광고 집행 전에도 내부 테스트 링크로 attribution 체계를 먼저 검증 가능
- 제품팀과 마케터가 같은 이벤트 이름을 기준으로 대화할 수 있음

## 자주 헷갈리는 포인트

### 1. `Referral`로 보이면 실패인가
아니다. UTM이 없거나, 세션 분류가 referral로 들어간 상황일 수 있다. 반드시 `세션 소스/매체`, `세션 캠페인`으로 직접 확인해야 한다.

### 2. 실시간에 안 보이면 실패인가
아니다. GA4는 반영 지연이 있을 수 있다. 실시간은 빠르지만 항상 표준 리포트와 정확히 동시에 갱신되지는 않는다.

### 3. 이전에 잘못 들어간 이벤트명은 어떻게 되나
과거에 잘못 저장된 이벤트명은 그대로 남는다. GTM 수정 후에는 새 이벤트부터 정상 이름으로 들어온다.

## 운영 권장 루틴

### 마케터
매일 또는 캠페인 시작 직후 확인:
- `트래픽 획득`
- `실시간 개요`
- `탐색 -> UTM -> Event Conversion`

### 개발자/제품 담당
릴리즈 전후 확인:
- GTM Preview
- GA4 실시간 이벤트
- 내부 이벤트 로그(`utm_*`, `first_touch_*`, `last_touch_*`)
- 결제 webhook 반영 상태

## 최종 체크리스트
- [ ] UTM 링크를 시크릿 창에서 열었다
- [ ] `트래픽 획득`에서 `세션 소스/매체` 확인
- [ ] `트래픽 획득`에서 `세션 캠페인` 확인
- [ ] `실시간 개요`에서 핵심 이벤트 확인
- [ ] `탐색`에서 유입 -> 이벤트 전환 표 확인
- [ ] 내부 이벤트 로그에서도 attribution 필드 확인

## 결론
Parrotkit는 현재 UTM 기반 유입 분석을 시작할 준비가 되어 있다. GTM과 GA4, 내부 이벤트 저장 구조가 연결되어 있기 때문에, 지금부터는 링크를 어디에 어떻게 배포하느냐가 중요하다. 즉, 기술 세팅 단계는 거의 끝났고, 이제는 마케팅 실험 설계와 운영 습관이 성과 차이를 만든다.
