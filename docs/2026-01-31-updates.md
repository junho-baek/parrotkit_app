### 1. 배포 환경 결제 이슈 해결

#### 🐛 문제
- **증상**: 로컬 환경에서는 정상 작동하지만, Vercel 배포 환경에서 결제 화면으로 이동하지 않음
- **사용자 보고**: "배포 환경에서는 결제 화면으로 안 넘어가는데, 원래 이런가?"

#### 🔍 원인 분석
- Vercel 배포 환경에 환경 변수가 설정되지 않음
- 로컬의 `.env.local` 파일은 배포 환경에 자동으로 반영되지 않음
- 특히 `NEXT_PUBLIC_VARIANT_PRO` (클라이언트 사이드 변수) 누락

#### ✅ 해결 방법
1. Vercel 대시보드 → Settings → Environment Variables
2. 다음 3개 환경 변수 추가:
   ```
   LEMONSQUEEZY_API_KEY=eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiJ9...
   LEMONSQUEEZY_STORE_ID=282768
   NEXT_PUBLIC_VARIANT_PRO=1263925
   ```
3. Vercel Redeploy 실행
4. 배포 환경에서 결제 플로우 정상 작동 확인 ✅

**교훈**: 환경 변수는 로컬과 프로덕션 환경에서 별도 설정 필요

## 🔧 기술적 의사결정

### 1. Migration 방식 선택

**선택지**:
- A) Drizzle Kit의 `generate` + `push` 사용
- B) 수동 SQL 스크립트 작성

**선택**: B (수동 SQL 스크립트)

**이유**:
1. Drizzle Kit 버전 이슈 (v0.20.18) - `push:pg` 명령어 오류
2. 환경 변수 인식 문제
3. 기존 테이블에 컬럼 추가이므로 간단한 ALTER TABLE로 충분
4. NeonDB 직접 접근이 더 빠르고 확실함

**구현**:
```typescript
// scripts/migrate.ts
const sql = neon(process.env.DATABASE_URL!);
await sql`
  ALTER TABLE mvp_users 
  ADD COLUMN IF NOT EXISTS subscription_id VARCHAR(255),
  ...
`;
```

### 2. Webhook userId 전달 방식

**선택지**:
- A) `custom_data.user_id` (snake_case)
- B) `custom_data.userId` (camelCase)

**선택**: B (camelCase)

**이유**:
1. JavaScript/TypeScript 컨벤션과 일관성
2. 프론트엔드와 백엔드 코드 스타일 통일
3. Lemon Squeezy API는 두 방식 모두 지원

### 3. 대시보드 구독 정보 조회 방식

**선택지**:
- A) localStorage에서 구독 정보 읽기
- B) API로 실시간 조회

**선택**: B (API 실시간 조회)

**이유**:
1. 보안: localStorage는 클라이언트에서 조작 가능
2. 실시간성: Webhook으로 업데이트된 최신 정보 반영
3. 정확성: DB가 Single Source of Truth

**구현**:
```typescript
// /api/user/profile 엔드포인트
const users = await db
  .select({
    subscriptionStatus: mvpUsers.subscriptionStatus,
    planType: mvpUsers.planType,
    // ...
  })
  .from(mvpUsers)
  .where(eq(mvpUsers.id, decoded.userId));
```

### 4. TypeScript Non-null Assertion 사용

**선택지**:
- A) Optional chaining + 기본값: `new Date(plan.saleEndDate || new Date())`
- B) Non-null assertion: `new Date(plan.saleEndDate!)`

**선택**: B (Non-null assertion)

**이유**:
1. 이미 `if (!plan.saleEndDate) return;`으로 null 체크 완료
2. 기본값을 제공하면 잘못된 날짜로 카운트다운될 위험
3. 타입 시스템에 정확한 제어 흐름 표현
